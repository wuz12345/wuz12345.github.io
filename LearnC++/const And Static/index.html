    简介¶

    C++的拷贝，赋值与移动语义在实际中是非常容易引起混乱的理解，因此，打算打算也一篇文章来详细介绍一下这三种情况，并给一些小的例子来方便理解。

    注：接下来要将的拷贝赋值和移动语义主要指在类中的使用

    拷贝构造函数¶

    区别于直接使用参数构造对象，拷贝构造是使用同一类的其他对象来构造对象。函数原型如下：

    ClassName (const ClassName &old_obj){
    th.s.attr = old_obj.attr;
}

    可以看到拷贝构造函数参数为对象引用，函数的执行过程是将旧对象的属性赋值给新对象。拷贝构造函数难点主要集中在以下几点：

    1. 为什么传引用做参数，传引用时如何区分const ClassName & obj和ClassName & obj？
    2. 除了使用对象构造对象时会显示调用拷贝构造函数外，还有什么情况也会调用拷贝构造函数？
    3. 为什么要区分深浅拷贝，深拷贝怎么做？
    4. 为什么某些类的要设计成noncopyable?

    引用传参与const引用¶

    为什么拷贝构造函数要传引用做参数呢？首先语法上构造函数不支持传值，以下代码会报错：

    class foo {
public:
    // ...
    foo(foo f){}
    // ...
};

    错误为copy constructor must pass its first argument by reference，问题的答案在于传值和传引用的区别，传值行为会需要对实参进行拷贝， 以确保函数行为不影响实参，因此在拷贝构造函数中传值，本身就会进行一次拷贝，而这次拷贝需要调用拷贝构造函数，这样就形成了对拷贝构造函数的递归调用， 因此拷贝构造函数传值是语法错误的。因此，拷贝构造函数只能传引用，这样的话为了避免拷贝过程中对参数的修改，const关键字就显得很有必要了
