    GDB调试

    启动与退出¶

    启动¶

    // test.cpp
#include <iostream>
#include <string>
int main(int argc, char **argv)
{
    int t =  stoi(argv[1]);

    for (int i = 0; i < t; i++)
    {
        std::cout << "i "
                  << "testing" << std::endl;
    }
    return 0;
}
    1. 最常用的:
    gdb your-program
    1. coredump:
    gdb your-program  coredump-file
    1. 调试运行中的程序:
    gdb your-program  pid

gdp -p pid  //直接写PID

    注: debug模式，gcc/g++ -g，cmake --config=debug

    1. 不输出信息:
    gdb --silent
    1. 传参数:
    gdb --args your-program your-program-args1 your-program-args2 ...

    退出¶

    quit [expression]
exit [expression]
q

    在gdb下运行程序¶

    编译¶

    上边提到过，调试的前提是要有调试信息，调试信息存储在.o文件里。使用-g编译。

    g++ -g test.cpp -o test

    以下面程序为例:

    int main(){
    return 0;
}

// .o 文件大小
-rw-r--r-- 1 root root  1368 Jau 17 09:52 nodebug.o
-rw-r--r-- 1 root root  2776 Jau 17 09:52 debug.o

// .out 文件大小
-rwxr-xr-x 1 root root 16464 Jau 17 09:54 nodebug.out
-rwxr-xr-x 1 root root 17400 Jau 17 09:54 debug.out

中间差值为调试信息，这就是debug模式和release模式的区别。
    使用objdump -g 可以查看.o文件的调试信息:

    // 在debug.o 中会看到 
Contents of the .debug_info section (loaded from debug.o):
Compilation Unit @ offset 0x0:
   Length:        0x4f (32-bit)
   Version:       4
   Abbrev Offset: 0x0
   Pointer Size:  8
   ... ... .... 
   ... ... ....

    启动程序¶

    编译完成后启动：

    1. 直接运行:
    $ gdb test

(gdb) run
    1. 在程序开始处(main)打断点：
    $ gdb test

(gdb) start

// Temporary breakpoint 4, main () at test.cpp:5
    1. 在操作系统执行用户进程开始处(一般在标准库当中)打断点:
    $ gdb test

(gdb) starti

// 0x00007fca67efc100 in _start () from /lib64/ld-linux-x86-64.so.2

    程序参数¶

    假设，程序名test参数10。

    // 启动gdb之前
gdb --args your-program your-program-args1 your-program-args2 ...
gdb --args test 10

// run 
(gdb) run args
(gdb) run 10


// set
(gdb) set args args1 args2 ...
(gdb) set args 10

    可以查看参数:

    (gdb) show args

    程序环境¶

    程序环境主要包括环境变量及其值,，修改和查看方式如下:

    (gdb) show environment [varname]

(gdb) set environment varname [=value]

(gdb) unset environment varname

    工作目录¶

    (gdb) set cwd [directory]

(gdb) show cwd

(gdb) cd [directory]

(gdb) pwd

    程序的输入和输出¶

    正常情况下，程序的输出终端与gdb终端一致，不过输入输出可以重定向:

    // 程序的输出被重定向到output
$ gdb test
(gdb) run 10 > output

    调试正在运行的程序¶

    // test.cpp 
// g++ -pthread -g test.cpp -o test

#include <iostream>
#include <string>
#include <thread>
#include <unistd.h>

static int times = 0;
bool isRuning = false;
void threadFunc()
{
    while (isRuning)
    {
        times += 1;
        sleep(3);
        std::cout << "test thread" << times << " times" << std::endl;
    }
    std::cout << "End test thread" << std::endl;
}
int main(int argc, char **argv)
{
    std::cout << "DEBUG: accessfile() called by process " << ::getpid() << " (parent: " << ::getppid() << ")" << std::endl;
    int time = std::stoi(argv[1]);
    isRuning = true;
    std::thread t(threadFunc);
    sleep(time);
    isRuning = false;
    t.join();
    return 0;
}

    使用attach来调试在gdb外运行的程序:

    (gdb) attach process-id

    上面命令如果出现：

    Could not attach to process.  If your uid matches the uid of the target
process, check the setting of /proc/sys/kernel/yama/ptrace_scope, or try
again as the root user.  For more details, see /etc/sysctl.d/10-ptrace.conf
ptrace: Operation not permitted.


sudo sysctl  kernel.yama.ptrace_scope=0

    杀掉子进程¶

    (gdb) kill

    同一个session调试多个程序inferior¶

    gdb中，一个终端对应一个session，一个进程对应一个inferior.

    查看当前session调试的进程信息:

    (gdb) info inferiors

    使用add-inferior和clone-inferior增加inferior，remove-inferiors用于减少inferior.

    因为gdb中，一个进程对应一个inferior，所以调试程序创建进程也会增加inferiors
