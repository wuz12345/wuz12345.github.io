    类模板

    类模板¶

    使用模板实现Stack¶

    #include <vector>
#include <cassert>

template<typename T>
class Stack {
  private:
    std::vector<T> elems;      // elements

  public:
    void push(T const& elem);  // push element
    void pop();                // pop element
    T const& top() const;      // return top element
    bool empty() const {       // return whether the stack is empty
        return elems.empty();
    }
};

template<typename T>
void Stack<T>::push (T const& elem)
{
    elems.push_back(elem);     // append copy of passed elem
}

template<typename T>
void Stack<T>::pop ()
{
    assert(!elems.empty());
    elems.pop_back();          // remove last element
}

template<typename T>
T const& Stack<T>::top () const
{
    assert(!elems.empty());
    return elems.back();       // return copy of last element
}

    类模板的声明¶

    和函数模板类似:

    template<typename T>  // template<class T>
   class Stack {
    //...
};

    和其他类型声明类似，T可以被用来声明成员和成员函数，在Stack这个例子中，T被用来声明元素类型，函数参数类型以及返回类型：

    template<typename T>
class Stack {
  private:
    std::vector<T> elems;      // elements

  public:
    void push(T const& elem);  // push element
    void pop();                // pop element
    T const& top() const;      // return top element
    bool empty() const {       // return whether the stack is empty
        return elems.empty();
    }
};

    和普通类不同，模板类的类型要包含模板参数<T>，例如Stack类的类型为Stack<T>，在类内使用时<T>可以省略，下面两段代码等价：

    template<typename T>
class Stack {
    //...
Stack (Stack const&);
Stack& operator= (Stack const&); 
};

template<typename T>
class Stack {
    //...
Stack (Stack<T> const&);
Stack<T>& operator= (Stack<T> const&); 
};

    成员函数的实现¶

    与非模板类的成员函数不同，模板类的成员函数在在类外实现时要使用完整类型<T>:

    template<typename T>
void Stack<T>::push (T const& elem)
{
    elems.push_back(elem); // append copy of passed elem }
}

    pop()函数实现如下:

    template<typename T>
T Stack<T>::pop ()
{
    assert(!elems.empty()); 
    T elem = elems.back(); // 保存
    elems.pop_back(); //删除
    return elem; //返回  return copy of saved element
}

    因为back()和pop_back()对于空容器存在未定义行为，所以断言。

    top()如下：

    template<typename T>
T const& Stack<T>::top ()const
{
    assert(!elems.empty());
    return elems.back(); // return copy of last element
}

    也可以在类内实现成员函数:

    template<typename T>
class Stack {
    void push (T const& elem) {
        elems.push_back(elem); // append copy of passed elem
    }
};

    使用Stack类¶

    与函数模板可以进行类型推断不同，使用类模板必须制定类型以实现模板实例化。 

    Stack<int>         intStack; // stack of ints 
Stack<std::string> stringStack; // stack of strings
// manipulate int stack 
intStack.push(7);
std::cout << intStack.top() << ’\n’;
// manipulate string stack
stringStack.push("hello");
std::cout << stringStack.top() << ’\n’;
stringStack.pop();
    只用调用成员函数时函数才会实例化，静态成员函数是当模板类使用时会实例化且只实例化一次。

    Partial Usage of Class Templates¶

    在模板类的定义中，往往需要实现多重操作例如++,--,<<等，此时就需要模板类型参数支持以上操作，在Stack<T>中增加打印函数，例如:

    template<typename T>
   class Stack {
    void printOn() (std::ostream& strm) const {
        for (T const& elem : elems) {
            strm << elem << ’ ’; // call << for each element
        }
    } 
};
    此时：

    Stack<std::pair<int,int>> ps;
ps.printOn(std::cout);  //Error std::pair<int,int>不支持`<<`操作
    但是对于其他函数则不报错，只有元素类型支持的操作在模板实例化后才不会报错(函数只有在调用的时候才会实例化)，因此以下使用是没有问题的。

    Stack<std::pair<int,int>> ps;
ps.push({4, 5});
ps.push({6, 7});
std::cout << ps.top().first << ’\n’; // OK
std::cout << ps.top().second << ’\n’; // OK

    Friends¶

    相较于上节提到printOn函数，更好的方法是重载<<：

    template<typename T>
class Stack {
    void printOn() (std::ostream& strm) const { 
        //...
    }
    // 为了使用私有变量 << 应该被声明为友元函数 非模板友元函数
    friend std::ostream& operator<< (std::ostream& strm, Stack<T> const& s) {
        s.printOn(strm);
        return strm;
    }
};

    对于模板友元函数的声明存在两种选择：

    1. 隐式的声明一个函数模板，使用不同的模板参数:

    template<typename T>
class Stack {
    //...
    template<typename U>
    friend std::ostream& operator<< (std::ostream&, Stack<U> const&);
};
    2. 前置声明类和函数:

    template<typename T>
class Stack;
template<typename T>
std::ostream& operator<< (std::ostream&, Stack<T> const&);

    之后，在声明该函数为友元:

    template<typename T>
class Stack {
    //...
    friend std::ostream& operator<< `<T>` (std::ostream&,
                                    Stack<T> const&);
};

    注意上述中<<后面的<T>，其实是将函数模板特化为类型<T>并将其声明为友元函数。

    类模板的特化(Specializations of Class Templates)¶

    可以使用特定的模板参数来特化类模板。不过，特化类模板时要求特化所有的成员函数。特化模板类时要使用template<>声明：

    template<>
class Stack<std::string> {
   // ...
};

    其成员函数特化：

    void Stack<std::string>::push (std::string const& elem)
{
    elems.push_back(elem); // append copy of passed elem 
}

    部分特化(偏特化,Partial Specialization)¶

    类是可以偏特化的。对于一个含有一个模板参数的类来说。一种典型的偏特化是将<T>声明为<T*>还是上面的Stack<>类，其可以偏特化为：

    template<typename T>
class Stack<T*> {
private:
    std::vector<T*> elems; // elements
public:
    void push(T*); // push element
    T* pop(); // pop element
    T* top() const; // return top element
    bool empty() const { // return whether the stack is empty
        return elems.empty();
    }
};

template<typename T>
void Stack<T*>::push (T* elem) // append copy of passed elem
{
    elems.push_back(elem);
}
template<typename T>
T* Stack<T*>::pop ()
{
    assert(!elems.empty());
    T* p = elems.back();
    elems.pop_back(); // remove last element
    return p; // and return it (unlike in the general case)
}
template<typename T>
T* Stack<T*>::top () const
{
    assert(!elems.empty());
    return elems.back(); // return copy of last element
}
    将模板特化为<T*>

    template<typename T>
class Stack<T*> {
    //...
};

    更常见的模板偏特化是多个模板参数T1,T2,...其中制定部分参数的类型:

    template<typename T1, typename T2>
class MyClass {
    //...
};

    两个模板参数特化为相同的类型: 

    template<typename T>
class MyClass<T,T> {
    //...
};

    指定第二个类型为int 

    template<typename T>
class MyClass<T,int> {
    //...
};

    特化为指针： 

    template<typename T1, typename T2>
class MyClass <T1*,T2*>{
    //...
};

    默认类模版参数(Default Class Template Arguments)¶

    在上面的提到的Stack<>类中，可以定义一个容器用来管理元素，并将此容器作为第二个模板参数，默认值为std::vector<>,

    #include <vector>
#include <cassert>

template<typename T, typename Cont = std::vector<T>> // 第二个模板参数，默认值为`std::vector<>`
class Stack {
  private:
    Cont elems;                // elements

  public:
    void push(T const& elem);  // push element
    void pop();                // pop element
    T const& top() const;      // return top element
    bool empty() const {       // return whether the stack is empty
        return elems.empty();
    }
};

template<typename T, typename Cont>
void Stack<T,Cont>::push (T const& elem)
{
    elems.push_back(elem);     // append copy of passed elem
}

template<typename T, typename Cont>
void Stack<T,Cont>::pop ()
{
    assert(!elems.empty());
    elems.pop_back();          // remove last element
}

template<typename T, typename Cont>
T const& Stack<T,Cont>::top () const
{
    assert(!elems.empty());
    return elems.back();       // return copy of last element
}
    • 存在两个模板参数时，定义成员函数时也要两个参数
    template<typename T, typename Cont>
void Stack<T,Cont>::push (T const& elem)
{
    elems.push_back(elem);     // append copy of passed elem
}

    在使用Stack<>时，如果只传入一个参数，第二个参数就会使用默认值:

    Stack<int> intStack; // Stack<int,std::vector<int>> intStack;

    Type Aliases(类型别名)¶

    typedefs Alias Declarations¶

    C++中可以使用两个关键字声明类型别名

    1. typedef
    typedef Stack<int> IntStack; // typedef
void foo (IntStack const& s); // s is stack of ints
IntStack istack[10]; // istack is array of 10 stacks of ints
    1. using
    using IntStack = Stack<int>; // alias declaration
void foo (IntStack const& s); // s is stack of ints
IntStack istack[10]; // istack is array of 10 stacks of ints

    注: 是别名，不是新类型

    Alias Templates(模板别名)¶

    template<typename T>
using DequeStack = Stack<T, std::deque<T>>;

    模板只能声明定义在全局或者在类中。

    成员类型的模板别名(Alias Templates for Member Types)¶

    和模板别名相同，给成员类型重新命名，方便使用：

    struct MyType {
    using iterator = ...; 
}
template<typename T>
using MyTypeIterator = typename MyType<T>::iterator;

    可以直接使用 

    MyTypeIterator<int> pos;

    而不是：

    typename MyType<T>::iterator pos;

    类型特征后缀 _t(Type Traits Suffix _t)¶

    C++14之后使用std::add_const_t<T>代替C++11的typename std::add_const<T>::type:

    namespace std {
    template<typename T> using add_const_t = typename add_const<T>::type;
}

    类模板参数推断(Class Template Argument Deduction)¶

    在C++17之前使用类模板需要传递所有的模板参数，从C++17开始，当构造函数能做类型推断时，不再需要传递所有参数:

    Stack<int> intStack1; // stack of strings
Stack<int> intStack2 = intStack1; // OK in all versions
Stack intStack3 = intStack1; // OK since C++17 Stack<>类的构造函数可以进行类型推断

    通过给构造函数提供初始参数，Stack<>类就会支持元素类型推断:

    template<typename T>
class Stack {
private:
    std::vector<T> elems; // elements
public:
    Stack () = default;
    Stack (T const& elem) // initialize stack with one element 含参构造函数
     : elems({elem}) {
    }
    //...
};

    C++17之后，以下语句，构造函数会进行类型推断 

    Stack intStack = 0; //模板参数被推断为 int，类型为Stack<int> 这个代码报错

Stack intStack(0); //这行可以

    上面的例子有两个需要注意的地方：

    1. Stack() = default;

    2. 构造函数参数为{elem}，注意花括号

    字符串常量的类模板参数推断(Class Template Arguments Deduction with String Literals)¶

    Stack stringStack = "bottom"; // Stack<char const[7]> deduced since C++17

    可以看到"bottom"被推断为char const[7],这是因为引用传参时类型不会衰变(decay)，这会带来一个问题：元素的大小被写死了。 但是，按值传参时类型会衰变为char const*，就不存在上面的问题了。因此，模板的定义可以修改为：

    template<typename T>
class Stack {
private:
    std::vector<T> elems; // elements
public:
    Stack (T const elem) // initialize stack with one element 含参构造函数
     : elems({elem}) {
    }
    //...
};

    为了避免拷贝，建议使用移动语义:

    template<typename T>
class Stack {
private:
    std::vector<T> elems; // elements
public:
    Stack (T const elem) // initialize stack with one element 含参构造函数
     : elems({std::move(elem)}) {
    }
    //...
};

    推断规则(Deduction Guides）¶

    可以看到构造函数按值传参会使string衰变为char*，在容器中处理原始指针会增加复杂度，因此一个比较好的办法是禁止将string推断为char*。使用自定义的推断规则即可：

    Stack(char const*) -> Stack<std::string>;

    模板化聚合(Templatized Aggregates)¶

    聚合类也可以模板化:

    template<typename T>
struct ValueWithComment {
    T value;
    std::string comment;
};

ValueWithComment<int> vc;
vc.value = 42;
vc.comment = "initial value";

    也可以为其增加推断规则:

    ValueWithComment(char const*, char const*) -> ValueWithComment<std::string>;

ValueWithComment vc2 = {"hello", "initial value"}; // 如果没有推断规则，本语句非法，`ValueWithComment`为提供构造函数
