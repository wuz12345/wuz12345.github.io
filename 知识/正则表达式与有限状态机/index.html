    背景¶

    最近在复习正则表达式的一些知识时，有了一些新的发现，写本篇blog的动机来自于下面这句话：

    • Regular expressions describe patterns which can be recognized by finite state machines (FSM)

    正则表达式描述的模式可以等价为一个有限状态机。初看这句话一脸懵逼，等等，正则表达式我会啊，有限状态机，我也会啊，这俩等价？我不到啊！为了解决初看到这句话的疑问，便沿着这个方向重新看了一下有限状态机与正则表达式。 通过本篇博客您会知道以下知识：

    1. 什么是快乐星球？不，什么是正则表达式，怎么看懂这玩意？
    2. 什么是有限状态机？这个只要知道概念就能自然看懂。
    3. 这两个如何互相转换，最重要的，如何像推导数学公式一样，完成转换。

    Let’s go!!

    正则表达式¶

    正则表达式最早是1951年由数学家Stephen Cole Kleene提出，从1968年开始到现在，正则表达式用途主要有两种： 1. 模式匹配; 2. 词法分析。 第一个正则表达式的实现是由C语言的作者完成的，之后在上世纪70年代他又将其引入Unix操作系统。Linux上面的grep程序(Global search for Regular Expression and Print matching lines)就是正则表达式在模式 匹配中最著名的应用。

    正则表达式的语法¶

    我们熟悉的正则匹配模式其表现往往是一串字符串，比如我们熟悉的*表示通配符。每一个正则表达式模式字符串中的字符都分为两种，一种是元字符metacharacter，一种是正则字符regular character。

    正则表达式的主要语法是通过元字符的语法规则实现的，下表总结了metacharacter的语法规则：

    Syntax Description
    ^ 匹配字符串的起始位置. 在基于行的工具中匹配行的开头
    . 匹配任意单字符,a.c中.可以用任意字符串替换，但是[a.c]只匹配a、.、c
    [] 匹配括号中的单字符,[abc]匹配a、b、c
    - 这个字符在
    [^ ] 是[]的 非 操作，匹配不含括号中单字符的字符串
    $ 匹配结尾符
    ( ) 可以将括号内的字符串看作一个整体进行匹配
    * 匹配>=0个重复的在*号之前的字符。
    {m,n} 匹配m到n次个重复的在{m,n}号之前的字符。
    ? 或操作，在符号前的字符出现一次或不出现
    + 比*多匹配一次
    | 或运算符，匹配符号前或后的字符.

    熟记上面的规则，就能很好的使用正则表达式了用来进行字符串匹配。等等？？这就完了？？这岂不是啥也没说。这个状态机有啥关系？？

    正则表达式和状态机的关系在于，计算机对正则表达式的处理。我们学会正则表达式后，能根据需求去定义自己的正则表达式，当我们将自己定义的正则表达式输入进计算机，计算机是如何处理的呢？计算机是怎么知道我们要做什么呢？？没错计算机要做的就是将我们定义的正则表达式翻译成一个状态机。如何能够将正则表达式翻译为状态机呢？Thompson’s construction就是将正则表达式翻译成有限自动机的算法。接下来我们看一下这个算法是如何实现这一功能的。
