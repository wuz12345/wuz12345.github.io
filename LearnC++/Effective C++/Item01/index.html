    Item1 理解模版类型推断¶

    下面的伪代码展示了模版类型的声明与定义部分：

    template <typename T>
void f(ParamType param)
f(expr)

    类型推断指的是在使用上面的代码f(expr)调用函数f时，编译器如何通过表达式(expr)的类型推断模版类型T和参数类型(ParamType).比如，下面这段代码：

    template <typename T>
void f(const T& param)

int x=0;
f(x)

    因为，x is int因此模版类型和参数类型为T is int paramType is const T&。模版类型推断主要分为以下三种情况：

    1. ParamType是指针或引用类型，不是Universal reference；
    2. ParamType是Universal reference；
    3. ParamType不是指针或引用，函数按值传参。

    注：Universal reference 类似于右值引用类型，和移动语义相关，其形式一般为T &&

    上面三种情况的具体规则如下：

    case1:参数类型是普通指针和引用¶

    这种情况下的推断规则非常简单，一句话:忽略指针与引用，保留const。

    参数类型为引用¶

    参数类型为引用，其参数类型推断如下： 

    template <typename T>
void f(T& param)
int x = 2;
const int cx = x;
const int &rc = x;

    1. f(x)，T is int, ParamType is int&;
    2. f(cx), Tis const int, ParamType is const int&(保留const);
    3. f(rx), Tis const int, ParamType is const int&(忽略引用);

    参数类型为const引用¶

    参数类型为const引用，其参数类型推断如下： 

    template <typename T>
void f(const T& param)
int x = 2;
const int cx = x;
const int &rc = x;

    1. f(x)，T is int, ParamType is const int&;
    2. f(cx), Tis int, ParamType is const int&(忽略变量声明中的const，使用参数类型声明的const);
    3. f(rx), Tis int, ParamType is const int&(忽略引用);

    参数类型为指针¶

    参数类型为const引用，其参数类型推断如下： 

    template <typename T>
void f(T* param)
int x = 2;
const int *px = &x;

    1. f(&x)，T is int, ParamType is int*;
    2. f(px), Tis const int, ParamType is const int*(使用参数类型声明的const);

    总结：case1的情况下推断模板参数类型T时，忽略引用与指针，const要看参数类型，如果参数类型有const，变量定义为const则T不是const.

    case2:参数类型是Universal Reference¶

    当参数类型被声明为T&&时，其推断规则如下：

    1. 如果expr是左值，则参数类型和模板类型推断为左值引用；
    2. 如果expr是右值，参照case1。

    左值和右值稍后介绍，左值可以简单理解为可以取地址的值，与此相对不能取地址的值是右值。

    template <typename T>
void f(T&& param)
int x = 2;

const int cx =x;
const int &rx = x;
    1. f(x)，x是左值，T is int& ParamType is int&(规则一)；
    2. f(cx)，T is const int& ParamType is const int&(规则一);
    3. f(rx)，T is const int& ParamType is const int&(规则一);
    4. f(27)，27是右值，T is int ParamType is int(规则二);

    case3:ParamType不是指针或引用¶

    按值传递的类型推断非常简单，其规则如下：

    1. expr是引用则忽略引用；
    2. expr是引用则忽略引用，忽略引用后如果被const修饰忽略const，同样忽略volatile
    template <typename T>
void f(T param)
int x = 2;

const int cx =x;
const int &rx = x;
    1. f(x)，T is int ParamType is int(规则一)；
    2. f(cx)，T is int ParamType is int(规则一);
    3. f(rx)，T is int ParamType is int(规则一);

    如果，按值传递指针时，const不能忽略

    template <typename T>
void f(T param)

const char* const ptr = "sssssss";
    1. f(ptr)，T is const char* ParamType is const char*

    其他情况下的类型推断¶

    Array参数¶

    Array类型在传参时实际传递的是首个元素的地址

    template <typename T>
void f(T param)

const char  name[] = "01234567";
    1. f(name) ，尽管name是array，但是T为const char*

    但是，在下面这种情况：

    template <typename T>
void f(T& param)

const char  name[] = "01234567";
    1. f(name) ，T为const char[8],参数类型为const char (&)[8]

    Function参数¶

    与Array相同，传递函数时使用的也是地址，会将函数指针作为参数。

    void func(int ,double)

    template <typename T>
void f(T param)

    调用f(func)，参数类型会被判断为void (*)(int,double)

    template <typename T>
void f(T& param)

    调用f(func)，参数类型会被判断为void (&)(int,double)
